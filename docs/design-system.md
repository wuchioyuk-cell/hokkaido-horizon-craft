# Hokkaido Horizon Design System

## Role Of This Document

This design system turns the brand direction into practical visual rules.

It does not authorize broad redesigns by itself. Before modifying code, always read this file together with:

- `AGENTS.md`
- `docs/design-bible.md`
- `docs/homepage-structure.md`

## Visual Goal

The site should feel like a Japanese editorial travel studio:

- Quiet
- Warm
- Human
- Minimal
- Authentic
- Premium but approachable
- Photography-led
- Spacious
- Local to Hokkaido

The interface should support the content and photography. It should never compete with them.

## Design Principles

- Less is more.
- Content before decoration.
- Photography before graphics.
- Calm before excitement.
- Trust before selling.
- Whitespace before density.
- Editorial rhythm before template symmetry.
- Useful detail before broad marketing claims.

## Color Language

Use quiet, natural tones that support photography.

Preferred color families:

- Ink / charcoal
- Soft white
- Snow white
- Cool off-white
- Muted stone
- Deep forest green
- Subtle blue-gray for snow, sky, and winter mood
- Very restrained warm neutrals only when needed

Avoid:

- Gold
- Beige-heavy palettes
- Neon accents
- Purple or blue SaaS gradients
- Heavy luxury-hotel colors
- Large decorative color washes

Color should improve readability, atmosphere, and hierarchy. It should not become decoration.

## Typography

Typography should be editorial and spacious.

Use:

- Large display typography for Hero and major emotional sections
- Clear sans-serif body text for readability
- Small uppercase labels with generous tracking
- Comfortable line height
- Short headings with strong rhythm

Avoid:

- Too many font styles
- Dense paragraph blocks
- Tiny low-contrast body text
- Startup-style headline stacks
- Decorative type that harms readability

Hero and major headings can be large, but smaller panels, navigation, and cards should stay compact and refined.

## Layout

Preferred:

- Full-screen or wide Hero composition
- Photography-led section openings
- Large whitespace
- Editorial asymmetry where useful
- Clean section transitions
- Strong mobile composition
- Clear reading widths
- Simple navigation and inquiry paths

Avoid:

- Dense card grids
- Floating cards inside cards
- Crowded columns
- Decorative blobs or abstract shapes
- Feature lists that feel like SaaS
- Repeated identical section patterns
- Product-catalog feeling on the homepage

## Photography

Photography is the primary visual language.

Preferred subjects:

- Hokkaido landscapes
- Roads and routes
- Snow and winter scenes
- Ski and resort access moments
- Seasonal nature
- Stays and arrival moments
- Local details
- Small human travel moments

Images should feel real and grounded. They should not feel like generic stock travel, luxury hotel fantasy, or staged influencer content.

When choosing between graphics and photography, choose photography unless a functional graphic is truly needed.

## Interface

The interface should be minimal and useful.

Use:

- Simple navigation
- Clear links
- Calm buttons
- Restrained borders
- Subtle section structure
- Minimal controls

Avoid:

- Heavy shadows
- Glowing buttons
- Overly rounded SaaS pills
- Dense icon rows
- Stats-heavy sales panels
- Banner-style promotions
- Unnecessary decorative UI

## Buttons And Links

Buttons should feel calm, confident, and practical.

Preferred:

- One primary action per major section
- Simple rectangular or lightly rounded shape
- Clear contrast
- Minimal hover state
- Short action labels

Avoid:

- Multiple competing CTAs
- Flashy hover effects
- Urgency language
- Discount-style buttons
- Luxury booking language

## Motion

Motion should be subtle.

Use:

- Slow opacity reveals
- Small vertical movement
- Gentle transitions
- Reduced-motion respect

Avoid:

- Bouncy animation
- Flashy entrance effects
- Parallax unless specifically approved
- Scroll tricks that make reading harder
- Motion that distracts from photography

## Content Density

The site should breathe.

Use fewer words when the image and heading already carry the meaning. Add detail only when it builds trust or helps a visitor make a decision.

Avoid long sales paragraphs on the homepage. Detailed service pages can carry more practical explanation, but still need calm spacing and readable rhythm.

## Accessibility And Mobile

- Text over images must remain readable.
- Headings must not overflow on small screens.
- Buttons must remain tappable.
- Navigation must remain clear.
- Mobile spacing should feel intentional, not cramped.
- Do not rely only on hover states.

## Never Introduce

These visual directions are out of bounds:

- Gold luxury style
- Heavy gradients
- SaaS landing page style
- Tech startup UI
- Travel agency banner style
- Fake luxury language
- Beige-heavy visual identity
- Generic AI landing page decorations

If a proposed visual change moves the site toward any of these, stop and explain the issue before implementing.
